# 疊池 · SUPERPOOL

> 一款以《後室》(The Backrooms)為根、探討平行宇宙的 3D 網頁類遊戲體驗。
> 你 noclip 進的不是「某個地方的很多層」,而是**宇宙與宇宙之間的那層膜**;每往前一步,就關掉身後一個世界。

線上體驗前建議戴上耳機——聲音就是導航。

---

## 專案理念

在經典《後室》裡,恐怖來自「熟悉但錯位的場所」。本作把這個母題嫁接到**平行宇宙**上:

**你穿過的每一個房間,都是同一個房間在無數現實裡的一個版本。往前走,細節就悄悄變動;而水面倒影,是一個「幾乎正確、卻微妙錯誤」的平行分支。**

池核(poolcore)在這裡不是裝飾,是主機制——水的倒影天生就是平行世界的視覺隱喻。整段體驗其實是**走在「分岔」本身裡面**。越往深處走,遊戲越是在問一個莊子式的問題:「家」到底存在過,還是只是你剛好起步時分岔最低的那個分支?結局不給「你回家了」——當你真的把分岔度壓到接近零,揭露的是這只是又一個分支,「原初的家」從不存在。以《齊物論》的胡蝶夢作結。

這不是要嚇你的遊戲,是一個讓你**在無盡的近似房間裡,慢慢接受回不去**的遊戲。恐怖來自「不對勁的持續累積」,不是突發的巨響。

### 五根設計支柱

1. **倒影即平行宇宙** — 水面倒影永遠 render 一個**不同 seed** 的分支,而非反射現實。全作的 signature。
2. **不可逆的分岔** — 門只會把你帶進新分支,永遠回不到出發的宇宙。每一步都關掉一個世界。
3. **可讀線索的選擇** — 讀微小線索、決定靠近哪道門。把「走廊漫遊」變成「遊戲」的核心迴圈。
4. **檔案感(found-footage)** — 1990 年代、VHS 顆粒、掃描線、色差、REC 時間戳(一個不存在的日期)。
5. **聲音即導航** — 嗡鳴走音程度、殘響、房間音隨分岔度變化。用耳朵就能判斷離家多遠。

## 技術棧

Next.js 15(App Router)· React Three Fiber · TypeScript · Zustand · Web Audio API · PWA。

```bash
npm install
npm run dev        # 開發伺服器 localhost:3000
npm run build      # 正式建置(型別檢查含在其中)
```

## 架構

### 決定論地基(最重要的一條)

全作的核心不變式:**`(seed, D) → room` 是純函數**(`src/core/room.ts`,由 `mulberry32` PRNG 驅動)。這是三個功能的共同前提,任一崩了三個都崩:

1. **倒影** — 水面 render `generateRoomSpec(mirrorSeed(branchId), …)`,一個**不同但可重現**的宇宙。
2. **重播與分享** — 一整段旅程完全由 `path[]`(門的序號序列)從固定起點決定;`src/core/branch.ts` 負責重播。把 path 編進 `?b=` 連結,任何人打開就重現同一條分岔。
3. **impossible-space** — 穿門時抽換房間內容並把玩家傳回原點;記憶體永遠只需約兩個房間,體感卻無盡。

> ⚠️ 修改 `generateRoomSpec` 時**絕不能重排 `rng()` 呼叫順序**——抽取順序是承重結構,`replayPath` 靠重跑同一串 rng 還原門的分岔值,任何插入/刪除都會悄悄讓所有分享連結失效。

### 分岔模型(D)

一個純量 `D ∈ [0,1]`,代表「離家有多遠」,是全作中樞變數。每次穿門套用 `ΔD`(熵往外推,玩家選相干的門可對抗)。D 即時驅動:霧色、燈光密度、幾何錯位、水面擾動、嗡鳴走音、身影機率、原型權重、調色盤、後製強度。

### 元件樹

`Experience.tsx` 是組裝點(Canvas + 分岔迴圈 + 結局編排):

| 檔案 | 職責 |
|---|---|
| `Room.tsx` + `components/room/parts.tsx` | 依 `archetype`(5 種空間原型)純渲染 `RoomSpec`;`variant="mirror"` 與疊加態 `ghost` 共用同一元件 |
| `Pool.tsx` | signature 效果:把鏡像分支 render 進半解析度 FBO(反射相機 + 斜裁剪面,移植自 three.js `Reflector`),水面 shader 螢幕空間取樣 |
| `Player.tsx` | 拖曳環顧 + WASD + 虛擬搖桿、碰撞、穿門觸發、淹水室漂浮物理 |
| `Effects.tsx` | found-footage 後製:自訂 VHS tracking + 顆粒/色差/掃描線/暗角/Bloom,強度綁 D |
| `Hud.tsx` | REC / 時間戳 / TRACKING 讀數 / 結局字幕 / 分享按鈕 / 觸控 |

**狀態**:單一 Zustand store(`src/core/store.ts`)——`branchId`、`D`、`path[]`,與結局狀態機 `play → lucid(揭露)| super(疊加態)→ end(莊子收束)`。

**聲音**:`src/core/audio.ts` 是 module 級 Web Audio 單例(非 React),由使用者手勢啟動,元件透過 setter 推狀態進去。

**效能分級**:`src/core/quality.ts` 依裝置能力一次決定 dpr、FBO 解析度、後製 pass 數(手機自動降級)。

### 五種空間原型

每道門後的空間風格不同,由 seed 決定、D 偏壓封閉度:

- **poolhall 泳池廳** — 天窗、高窗、扶梯、滑水道。「家」永遠是這個。
- **arcade 拱廊水道** — 連續拱牆跨過水道,錯綜的縱深。
- **flooded 淹水室** — 整室在水中,慣性漂浮、悶音、牆面焦散光網。
- **storage 儲藏間** — 低天花板、成堆泳池椅、地面積水映倒影。
- **corridor 濕區廊道** — 窄長、排水槽、壁龕、假門。

## Debug 參數

| 參數 | 作用 |
|---|---|
| `?d=0.65` | 直接跳到任意分岔度 |
| `?a=arcade` | 強制指定原型 |
| `?b=021` | 重播一條分享的分岔(門序號序列) |
| `?touch=1` | 桌機強制開啟手機觸控 UI |

console 另有 `window.__superpool`(store)與 `window.__branch` 可直接操作。
